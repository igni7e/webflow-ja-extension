このファイルは、Chrome Web Storeに申請する際に使用する情報をまとめたものです。

---

## 基本情報

### 拡張機能名
```
Webflow管理画面日本語化
```

### 短い説明（132文字以内）
```
Webflowの管理画面を日本語化する拡張機能。デザイナー、エディター、CMS等のUIを自動翻訳。英語が苦手な方でもWebflowを快適に使えます。
```

### 詳細な説明

```
【Webflowを日本語で使いたい方へ】

Webflowは素晴らしいノーコードWebデザインツールですが、すべて英語表記のため、英語が苦手な方には使いづらい面があります。

この拡張機能は、Webflowの管理画面（Designer/Editor/Dashboard等）を自動的に日本語化し、デザイン作業に集中できる環境を提供します。

【主な機能】

✅ Webflow専用設計 - *.webflow.comドメインでのみ動作
✅ 226個以上の専門用語を翻訳 - Designer、Navigator、Style Panel等
✅ リアルタイム翻訳 - 動的に追加される要素も自動翻訳
✅ カスタマイズ可能 - CSVファイルで翻訳用語を自由に編集
✅ プライバシー重視 - 外部サーバーにデータを送信しません
✅ 軽量・高速 - ページ表示速度に影響なし

【翻訳される主な用語カテゴリ】

• デザインツール: Elements, Navigator, Style Panel, Interactions
• CMS機能: Collections, Assets, Dynamic Content
• レイアウト: Flexbox, Grid, Positioning, Spacing
• スタイリング: Typography, Colors, Effects, Transforms
• 公開・設定: Publishing, Hosting, SEO Settings, Custom Code
• アカウント: Dashboard, Team, Billing, Permissions

【使い方】

1. 拡張機能をインストール
2. Webflowの管理画面にアクセス
3. 自動的に日本語化されます

翻訳が適用されない場合は、ページを再読み込み（F5）してください。

【プライバシーとセキュリティ】

• 外部サーバーへのデータ送信なし
• 個人情報の収集なし
• オープンソース（コードは全て公開）
• Webflow専用動作（他のサイトには影響しません）

【カスタマイズ】

拡張機能フォルダ内の translation_terms.csv を編集することで、翻訳用語を追加・変更できます。

【サポート】

• バグ報告・機能要望: GitHub Issues
• オープンソース: https://github.com/yourusername/webflow-ja-extension

【免責事項】

この拡張機能は非公式プロジェクトであり、Webflow, Inc.とは一切関係ありません。

---
```

## Single Purpose（単一目的）の説明

Chrome Web Storeの審査では、拡張機能の「単一目的」を明確にする必要があります。

**この拡張機能の単一目的:**
```
Webflowの管理画面UIテキストを英語から日本語に翻訳すること
```

**詳細説明:**
```
この拡張機能は、Webflow（*.webflow.com）のウェブページ上に表示される英語のUIテキストを、
事前に定義された用語集（CSVファイル）に基づいて日本語に自動置換することのみを目的としています。

その他の機能は一切含まれておらず、翻訳以外の目的では使用されません。
```

## 審査時の注意事項

### 必須確認項目

1. ✅ **プライバシーポリシーの公開**
   - PRIVACY_POLICY.mdの内容をGitHub PagesまたはWebサイトで公開
   - manifest.jsonの"homepage_url"に追加

2. ✅ **manifest.jsonの"permissions"説明**
   - storage: 将来的なユーザー設定保存用（現在未使用）
   - host_permissions: Webflowサイトでの翻訳機能実行用

3. ✅ **Single Purpose**
   - UIテキストの翻訳のみを目的としている
   - その他の機能は含まない

4. ✅ **最小権限の原則**
   - 必要最小限の権限のみを要求
   - *.webflow.comドメインにのみアクセス

5. ✅ **データの取り扱い**
   - 外部サーバーへのデータ送信なし
   - ユーザーデータの収集なし
   - すべての処理はローカルで完結

### よくある拒否理由とその対策

**拒否理由1: プライバシーポリシーが不明確**
→ 対策: PRIVACY_POLICY.mdを詳細に記載し、公開URL���manifest.jsonに追加

**拒否理由2: 権限の説明不足**
→ 対策: 各権限の使用目的を明確に説明

**拒否理由3: スクリーンショット不足**
→ 対策: 最低3枚、推奨5枚のスクリーンショットを用意

**拒否理由4: 機能の説明不足**
→ 対策: 詳細な説明文とREADMEを用意
